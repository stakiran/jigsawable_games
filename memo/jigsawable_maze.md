# Jigsawable Maze — 設計ドキュメント

## コンセプト

ジグソアブルな迷路ゲーム。PC ブラウザ (HTML/JS) で動作する、巨大な1つの迷路を何日もかけて踏破する体験。

## 設計方針

### 迷路の構造

- **グラフベース**：グリッドではなく、ノード（点）とエッジ（線）で構成
- **不均一配置**：ノードはポアソンディスク分布でランダムに配置。エッジの長さもバラバラ
- 隣接関係はドロネー三角分割で構築し、その上にランダム全域木（ウィルソンのアルゴリズム）を生成
- **全域木のエッジのみが通行可能**。画面に表示されるエッジ＝行き来できる道、という原則を守る

### 迷路パラメータ（現在値）

- ワールドサイズ: 3000×3000（仮想座標）
- ポアソンディスク最小距離: 45 → 約 2,500〜3,000 ノード
- スタート: 左上に最も近いノード
- ゴール: 右下に最も近いノード

### 表示

- **探索モード**: プレイヤー中心にカメラ追従。周辺のノードとエッジのみ描画。ビネット（周辺減光）あり
- **俯瞰モード**: 全体を縮小表示。進捗の全体像が見える
- **Tab キーで切り替え**
- ミニマル・抽象的な見た目。暗い背景に点と線

### 足跡（進捗の可視化）

- 訪問済みノードとエッジは紫系の色で染まる
- 新しい足跡ほど明るく、古い足跡は徐々に暗くなる（ステップ番号ベースの経年変化）
- ヘッダーに探索率（訪問ノード数/全ノード数）と歩数を表示
- 画面下部にプログレスバー

### 操作

- **X キー**: 方向転換（隣接ノードを順に選択）
- **Z キー**: 選択中のノードへ移動
- **Tab キー**: 探索/俯瞰モード切り替え
- 隣接ノードは角度順にソートされており、X で時計回りに巡回する

### カーソル表示

- **現在位置ノード**: 白い光る円（グロー付き）
- **選択中の移動先ノード**: 白い光る円（やや大きめ、グロー付き）
- **選択エッジ**: 明るい白線で強調
- **アローヘッド**: 選択エッジの60%地点に白い三角矢印を表示。現在位置→移動先の方向を示す
- 非選択の隣接ノード/エッジは薄く表示

### 永続性（セーブ/ロード）

- localStorage に保存。ブラウザを閉じても再開可能
- 保存データ: ノード座標、全域木の隣接リスト、訪問ステップ配列、プレイヤー位置、歩数、訪問数、完了フラグ
- セーブキー: `jigsawable_maze_v4`
- UI ボタン: 保存 / 読込 / 新規

## 技術スタック

- 単一 HTML ファイル（CSS・JS インライン）
- Canvas 2D で描画
- フォント: DM Mono（Google Fonts）
- 外部ライブラリなし

## アルゴリズム概要

### 1. ポアソンディスク分布

ワールド空間に最小距離 `r` を保ってランダムにノードを配置。Bridson のアルゴリズム（k=30 試行）。

### 2. ドロネー三角分割（Bowyer-Watson）

全ノードに対して三角分割を行い、自然な隣接関係（辺）を抽出。スーパートライアングルの頂点は除外。

### 3. ウィルソンのアルゴリズム（ランダム全域木）

ドロネーの隣接グラフ上でループ消去ランダムウォークにより、均一分布の全域木を生成。この木のエッジ集合が迷路の通路となる。
